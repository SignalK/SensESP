55 // Provides one parameter reading from the combination sensor.
56 // val_type determines which particular parameter is output.
57 // Note that since the filter algorithm works best when all 9-DOF
58 // parameters are captured simultaneously, and at a specified
59 // constant rate, there is only a single call to
60 // sensor_fxos_fxas->gatherOrientationDataOnce() in this method.
61 // Arbitrarily, this update happens when the compass heading
62 // is requested; all the remaining parameters are retrieved
63 // by calling the appropriate sensor_fxos_fxas->getter method.
64 // If the compass heading parameter is not the one that is set
65 // to the fastest repetition rate (in main.cpp when calling
66 // Read9DOF::Read9DOF()), then you should move the
67 // gatherOrientationDataOnce() call into the case block
68 // for whichever parameter _is_ updated the fastest - this
69 // ensures that that parameter and all others are never stale.
